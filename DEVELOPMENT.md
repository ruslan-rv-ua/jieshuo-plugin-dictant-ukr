# DEVELOPMENT.md

Інструкції для розробника плагіну "Диктант" для Jieshuo.

## Збірка

Потрібен Python 3.11+ (для вбудованого `tomllib`).

```bash
python build.py
```

Результат: `dist/диктант-{version}-uk.ppk` — звичайний zip-архів з `src/main.lua` всередині.

Для зміни версії — редагуй `project.toml` і оновлюй рядок `-- version:` у шапці `src/main.lua`.

## Встановлення на пристрій

1. Перенеси `.ppk`-файл на Android-пристрій.
2. У Jieshuo: Меню → Плагіни → Встановити → обери файл.

## Структура проєкту

```
src/main.lua      # весь код плагіну (єдиний файл — обмеження Jieshuo)
build.py          # скрипт збірки
project.toml      # метадані: name, version, language, build_name
```

## Обмеження платформи

Jieshuo завантажує плагін як один файл `main.lua`. Стандартний `require` для власних модулів не працює. Весь код — в одному файлі.

Доступні глобальні об'єкти, які Jieshuo інжектує в контекст:

| Змінна | Що це |
|---|---|
| `this` | Android `Context` |
| `activity` / `service` | хост-об'єкт Jieshuo |
| `node` | поточний вузол доступності (focused node) |
| `require "import"` + `import "..."` | міст до Java/Android API |

Корисні методи `service`:
- `service.isEditView(node)` — чи фокус на полі вводу
- `service.getText(node)` — поточний текст поля
- `service.setText(node, text)` — встановити текст
- `service.speak(text)` — озвучити текст (перериває поточне озвучення)
- `service.appendSpeak(text)` — озвучити текст у черзі

## Налаштування

На початку `src/main.lua` є секція налаштувань:

```lua
local CLEAR_EDITOR_PHRASE = "диктант"   -- команда "очистити редактор"
local CAPITALIZE_PHRASE = "з великої"   -- команда "наступне слово з великої"
local VIBRATE = true                    -- вібрація при активації
```

## Пайплайн обробки тексту

Розпізнаний текст проходить наступні кроки по порядку:

1. **`decodePunctuations()`** — заміна слів-пунктуацій на символи (`крапка` → `.`, тощо) та `CAPITALIZE_PHRASE` на внутрішній символ `~`
2. **Склейка з попереднім текстом** — або очищення, якщо текст починається з `CLEAR_EDITOR_PHRASE`
3. **`formatPunctuations()`** — видалення пробілів перед `.`, `,`, `?`, `!`, `:`
4. **`stripLines()`** — обрізання пробілів на початку/кінці кожного рядка
5. **`capitalize()`** — велика літера після `.`, `?`, `!` та після символу `~`; видалення `~`

## Потік виконання

Скрипт виконується одразу при активації плагіну (немає точки входу типу `main()`):

```
активація → вібрація → перевірка isEditView → startListening()
                                    ↓ ні
                             speak("Не в редакторі")
```

`startListening()` реєструє `RecognitionListener` і запускає Android Speech Recognizer. Результат обробляється в колбеці `onResults`.
